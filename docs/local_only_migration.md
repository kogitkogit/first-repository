# 로컬 전용 전환 계획 (Capacitor SQLite)

> 목표: 현재 UI/기능을 유지한 채 서버 의존을 제거하고, 로컬 SQLite로 동작하게 전환한다.
> 기준일: 2026-01-23

---

## 1) 전환 원칙

- 로그인/회원가입 화면은 유지하되 **로컬 사용자로 자동 진입**한다.
- 모든 데이터는 **Capacitor SQLite**에 저장한다.
- 서버(FastAPI)와 Postgres는 사용하지 않는다.
- UI/문구는 기존 화면을 최대한 유지한다.

---

## 2) 작업 단계(계획)

1. 현행 API 의존도 분석 및 기능별 데이터 흐름 매핑
2. 로컬 DB 설계(테이블 스키마, 인덱스, 마이그레이션 버전 정책)
3. 로컬 DB 모듈 골격 작성(연결/초기화/마이그레이션)
4. 로컬 DB 접근 레이어 구현(CRUD 유틸, 트랜잭션, 공통 변환)
5. 기능별 전환 1차: 차량/주행거리/정비/주유/소모품
6. 기능별 전환 2차: 타이어/법적정보/알림/대시보드 요약
7. 로그인/세션 로직 제거 및 로컬 사용자 자동 진입 흐름 반영
8. 오프라인 안정성/에러 처리/데이터 검증 보강
9. 마이그레이션 테스트 + 샘플 데이터 시드 + 화면 점검
10. 백업/복구(선택) 및 최종 QA
