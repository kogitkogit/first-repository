# CarCare Play Store 출시 체크리스트

기준일: 2026-04-07  
그래픽 자산 준비는 제외하고, 실제 제출과 심사 통과에 필요한 항목만 정리합니다.

## 1. 앱 빌드 / 패키징
- [ ] `web`에서 `npm run build` 성공
- [ ] `web`에서 `npm run cap:sync` 성공
- [ ] `web/android`에서 `gradlew.bat bundleRelease` 성공
- [ ] 산출물 확인: `web/android/app/build/outputs/bundle/release/app-release.aab`
- [ ] `versionCode` 증가 확인
- [ ] `versionName` 최종 반영 확인

## 2. 서버 / 운영 URL
- [ ] 운영 백엔드 HTTPS URL 확인
- [ ] Render 헬스체크 정상: `/api/health`
- [ ] 개인정보처리방침 공개 URL 확인
  - 권장 URL: `https://carcare-project.onrender.com/privacy-policy`
- [ ] 앱 내부 안내 문구와 실제 운영 상태 일치 확인

## 3. Play Console 앱 정보
- [ ] 앱 이름 확정
- [ ] 앱 카테고리 확정
- [ ] 연락처 이메일 입력
- [ ] 개인정보처리방침 URL 입력
- [ ] 광고 여부 선언
  - 현재 AdMob SDK가 없으면 `광고 없음`으로 맞추는 것이 안전
- [ ] 콘텐츠 등급 설문 작성
- [ ] 타겟 연령대 설정

## 4. Data Safety 초안
다음 정보는 수집한다고 보는 편이 안전합니다.

- [ ] 개인 정보
  - 이메일 또는 아이디
- [ ] 앱 활동 / 사용자 제공 정보
  - 차량번호
  - 차량 정보
  - 정비 / 주유 / 충전 / 비용 / 소모품 / 법적 서류 기록

다음 기준으로 답변 검토:
- [ ] 수집 목적: 앱 기능 제공
- [ ] 계정 관리 목적 포함 여부 확인
- [ ] 데이터가 전송 중 암호화되는지 확인
- [ ] 계정 삭제 요청 경로 확인

주의:
- 실제 푸시 알림이 아직 미구현이므로, 알림 데이터 처리나 광고 SDK 사용을 과장해 기재하면 안 됨

## 5. 심사용 앱 접근 정보
- [ ] 리뷰어가 로그인 가능한 테스트 계정 준비
- [ ] 회원가입 가능 여부 설명
- [ ] 차량 등록 후 어디를 보면 되는지 심사 안내 문구 준비

권장 안내 예시:
1. 앱 실행 후 회원가입 또는 제공된 테스트 계정으로 로그인
2. 차량 등록 화면에서 임의 차량 등록
3. 대시보드, 주유/충전, 정비, 비용 관리 패널 확인

## 6. 앱 내 정책 / 고지 상태
- [ ] 회원가입 화면 약관 `보기` 정상 동작
- [ ] 설정 화면에서 개인정보처리방침 확인 가능
- [ ] 설정 화면에 미구현 기능이 `준비 중`으로 명확히 표시됨
- [ ] 모바일 푸시 알림이 미구현이면 사용자에게 명확히 안내됨
- [ ] 백업 / 복구가 미구현이면 사용자에게 명확히 안내됨

## 7. 기능 QA
- [ ] 로그인
- [ ] 회원가입
- [ ] 차량 등록 / 삭제
- [ ] 초기 설정 가이드
- [ ] 주행거리 입력
- [ ] 주유 기록 추가 / 수정 / 삭제
- [ ] EV/PHEV 충전 기록 추가 / 수정 / 삭제
- [ ] 정비 기록 추가 / 수정 / 삭제
- [ ] 오일 / 필터 / 소모품 기록 저장
- [ ] 타이어 측정 / 교체 기록 저장
- [ ] 법적 서류 입력 / 이동
- [ ] 비용 관리 집계 확인
- [ ] 대시보드 카드 이동 확인

## 8. 모바일 실기기 점검
- [ ] 스크롤 위치 이상 없음
- [ ] 모달 중앙 정렬 정상
- [ ] 키보드가 입력창을 가리지 않음
- [ ] 하단 NAV와 입력 버튼이 겹치지 않음
- [ ] 작은 화면에서 버튼 / 배지 줄바꿈 문제 없음

## 9. 출시 전 최종 정리
- [ ] 불필요한 로그 / `__pycache__` / 빌드 산출물 정리
- [ ] 최신 코드 commit / push
- [ ] Render 재배포 확인
- [ ] Supabase 스키마 반영 상태 확인
- [ ] AAB 최종본 재생성

## 10. 현재 남은 실제 리스크
- [ ] 푸시 알림 미구현
- [ ] 백업 / 복구 미구현
- [ ] 플레이스토어용 그래픽 자산 미준비
- [ ] 심사자용 테스트 계정 / 안내 문구 미확정

## 11. 현재 기준 제출 가능 판단
- 그래픽 자산만 제외하면, 정책 문구와 공개 개인정보처리방침 URL 정리가 끝난 뒤 `내부 테스트 트랙 업로드`까지는 진행 가능
- 프로덕션 제출 전에는 `Data Safety`, `앱 접근 정보`, `광고 여부`, `실기기 QA`를 반드시 마무리해야 함
