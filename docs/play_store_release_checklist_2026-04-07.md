# 내차수첩 Play Store 출시 체크리스트

기준일: 2026-04-13  
그래픽 자산 준비는 제외하고, 실제 제출과 심사 대응에 필요한 항목만 정리합니다.

## 1. 빌드 / 패키징
- [ ] `web`에서 `npm run build` 성공
- [ ] `web`에서 `npm run cap:sync` 성공
- [ ] `web/android`에서 `gradlew.bat bundleRelease` 성공
- [ ] 제출물 확인: `web/android/app/build/outputs/bundle/release/app-release.aab`
- [ ] `versionCode` 증가 확인
- [ ] `versionName` 최종 반영 확인

## 2. 운영 URL
- [ ] 운영 백엔드 HTTPS URL 확인
- [ ] Render 헬스체크 정상: `/api/health`
- [ ] 개인정보처리방침 공개 URL 확인
  - 권장 URL: `https://내차수첩-project.onrender.com/privacy-policy`
- [ ] 앱 내 정책 링크와 실제 공개 URL 내용 일치 확인

## 3. Play Console 스토어 정보
- [ ] 앱 이름 확정
- [ ] 앱 카테고리 확정
- [ ] 개발자 연락처 이메일 입력
  - Google Play는 앱 게시/업데이트 시 연락처 이메일을 요구함
- [ ] 개인정보처리방침 URL 입력
- [ ] 광고 여부 선언
  - AdMob를 넣을 예정이면 `Contains ads = Yes`
- [ ] 콘텐츠 등급 설문 작성
- [ ] 타깃 연령대 설정

## 4. Data Safety
다음 데이터 처리를 중심으로 검토합니다.

- [ ] 개인 정보
  - 계정 식별 정보(아이디)
- [ ] 앱 활동 / 사용 정보
  - 차량번호
  - 차량 정보
  - 정비 / 주유 / 충전 / 비용 / 소모품 / 법적 서류 기록
- [ ] 광고 또는 마케팅
  - AdMob 사용 시 광고 ID, 앱 세트 ID, IP 주소, 진단 정보, 광고 상호작용 정보 처리 여부 반영

필수 확인:
- [ ] 수집 목적: 앱 기능 제공
- [ ] 계정 관리 목적 포함 여부 확인
- [ ] 광고/분석 목적 포함 여부 확인
- [ ] 데이터 전송 중 암호화 여부 확인
- [ ] 계정 삭제 요청 경로 확인

주의:
- AdMob를 넣는 경우 Google Mobile Ads SDK의 데이터 공개 요구사항을 반영해야 함
- 실제 미구현 기능을 Data Safety에 과장해서 적으면 안 됨

## 5. 심사 대응용 앱 접근 정보
- [ ] 리뷰어가 로그인 가능한 테스트 계정 준비 또는 비회원 시작 가능 여부 확인
- [ ] 회원가입/비회원 시작 흐름 설명 준비
- [ ] 차량 등록 후 어떤 기능을 확인하면 되는지 심사 안내 문구 준비

권장 안내 예시:
1. 앱 실행 후 약관 동의
2. `회원가입해서 시작하기` 또는 `비회원으로 시작하기` 선택
3. 차량 등록
4. 대시보드, 주유/충전, 정비, 비용 관리 기능 확인

## 6. 정책 / 고지 상태
- [ ] 회원가입 화면의 약관 `보기` 동작 확인
- [ ] 설정 화면에서 개인정보처리방침 확인 가능
- [ ] 준비 중 기능은 `준비 중`으로 명확히 표시
- [ ] 푸시 알림 미구현 상태를 사용자에게 과장 없이 안내
- [ ] 백업/복구 제한 사항을 사용자에게 명확히 안내
- [ ] AdMob 적용 시 UMP 또는 동등한 사용자 동의 흐름 준비

## 7. 기능 QA
- [ ] 로그인
- [ ] 회원가입
- [ ] 차량 등록 / 삭제
- [ ] 초기 설정 가이드
- [ ] 주행거리 입력
- [ ] 주유 기록 추가 / 수정 / 삭제
- [ ] EV/PHEV 충전 기록 추가 / 수정 / 삭제
- [ ] 정비 기록 추가 / 수정 / 삭제
- [ ] 오일 / 필터 / 소모품 기록 저장
- [ ] 타이어 측정 / 교체 기록 저장
- [ ] 법적 서류 입력 / 이동
- [ ] 비용 관리 집계 확인
- [ ] 대시보드 카드 이동 확인

## 8. 모바일 기기 QA
- [ ] 스크롤 위치 이상 없음
- [ ] 모달 중앙 정렬 정상
- [ ] 키보드가 입력창을 가리지 않음
- [ ] 하단 NAV와 입력 버튼이 겹치지 않음
- [ ] 작은 화면에서 버튼 / 배지 줄바꿈 문제 없음

## 9. 출시 전 최종 정리
- [ ] 불필요한 로그 / `__pycache__` / 빌드 산출물 정리
- [ ] 최신 코드 commit / push
- [ ] Render 재배포 확인
- [ ] Supabase 스키마 반영 상태 확인
- [ ] AAB 최종본 생성

## 10. 현재 남은 실제 리스크
- [ ] 실제 푸시 알림 미구현
- [ ] 백업 / 복구 전용 기능 미구현
- [ ] 플레이스토어용 그래픽 자산 미준비
- [ ] 개발자 연락처 이메일 실제 값 미확정

## 11. 현재 기준 출시 판단
- 그래픽 자산을 제외하면 내부 테스트 트랙 업로드까지는 진행 가능한 상태입니다.
- 프로덕션 제출 전에는 `Data Safety`, `앱 접근 정보`, `광고 여부`, `실기기 QA`, `개발자 연락처 이메일`을 반드시 마무리해야 합니다.
