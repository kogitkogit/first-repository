# 내차수첩 Google Play 이미지 정리

기준 출처:
- Google Play Console Help, `Add preview assets to showcase your app`
- Google Play Console Help, `Create and set up your app`

현재 업로드용으로 정리한 폴더:
- `app_icon/`
- `feature_graphic/`
- `phone_screenshots/`

## 1. 필수 이미지

### 앱 아이콘
- 파일 위치: `app_icon/`
- 권장 파일명: `app-icon-512.png`
- 규격:
  - `512 x 512 px`
  - `PNG`
  - 최대 `1024 KB`
- 메모:
  - 실제 앱 아이콘과 동일 계열로 유지
  - 순위, 가격, 배지 같은 문구 금지

### 기능 그래픽
- 파일 위치: `feature_graphic/`
- 권장 파일명: `feature-graphic-1024x500.png`
- 규격:
  - `1024 x 500 px`
  - `JPEG` 또는 `PNG 24-bit`
  - 알파 채널 없음 권장
- 메모:
  - 앱 핵심 가치만 짧고 강하게 전달
  - 아이콘과 똑같은 구성을 크게 반복하지 않기

### 휴대전화 스크린샷
- 파일 위치: `phone_screenshots/`
- 권장 파일명:
  - `01-home.png`
  - `02-fuel.png`
  - `03-maintenance.png`
  - `04-tire.png`
  - `05-costs.png`
- 규격:
  - `JPEG` 또는 `24-bit PNG`
  - 각 이미지 최대 `8 MB`
  - 짧은 변 `320 px` 이상
  - 긴 변 `3840 px` 이하
  - 비율은 휴대전화 실제 화면 기준 유지
- 최소 수량:
  - 휴대전화 스크린샷 `2장 이상`

## 2. 권장 스크린샷 구성

내차수첩은 아래 5장을 기본 세트로 잡는 것이 좋음:

1. 대시보드
- 핵심 가치: 차량 상태 한눈에 확인

2. 주유/충전 관리
- 핵심 가치: 연료비, 충전비, 기록 관리

3. 정비 이력
- 핵심 가치: 정비 기록 저장과 관리

4. 타이어 관리
- 핵심 가치: 공기압/트레드/교체 이력 추적

5. 비용 관리 또는 법적 서류
- 핵심 가치: 유지비와 필수 문서 관리

## 3. 문구 가이드

스크린샷 상단 짧은 문구 예시:

- `차량 상태를 한눈에`
- `주유와 충전 기록을 간편하게`
- `정비 이력을 놓치지 않게`
- `타이어 상태까지 꼼꼼하게`
- `유지비와 서류까지 한 곳에서`

주의:
- 텍스트가 너무 많으면 안 됨
- 앱에서 실제 제공하지 않는 기능 문구 금지
- 알림, 백업, 복구, 광고 관련 표현은 실제 기능과 일치해야 함

## 4. 현재 상태

- `playstore_assets/` 안에 업로드 구조는 준비됨
- 실제 최종 이미지 파일은 아직 채워야 함

## 5. 다음 권장 작업

1. 실제 앱 화면 캡처 5장 선정
2. `phone_screenshots/`에 원본 저장
3. `feature_graphic/`에 1024x500 최종본 저장
4. `app_icon/`에 512x512 최종 아이콘 저장
5. Play Console 업로드 전 언어별 문구 재확인
